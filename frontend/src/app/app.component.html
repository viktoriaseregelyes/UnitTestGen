With this language, you can easily specify how tests should be generated for a given class or method.

  • CONSTRUCTOR – optional section, specifies the parameters of the constructor. Example: CONSTRUCTOR PARAM int id VALUE 1
  • MOCKING – optional section, definition of mock objects. Example: MOCK Service serviceMock
  • TEST – start of a new test case.
    • REPEAT – number of test repetitions.
    • METHOD – name of the method to be tested.
    • PARAM – method parameters and their values:
      • VALUE x – a single value
      • VALUES [x, y, z] – multiple values
    • VARIATION –method parameters that appear in tests with variable parameter values:
      • VALUES [x, y, z] – multiple values
      • VALUES FOR x TO y – numbers in sequence
    • EXPECT – expected output (not prefix for negation)
      • x – a single value
      • [x, y, z] – multiple values
      • FOR x TO y – numbers in sequence
    • EXPECT_EXCEPTION – expected exception, with optional message

Supported types example: int, float, double, char, boolean, string, enum, List<...>, Optional<...>, Map<...>, Set<...>, custom type.
